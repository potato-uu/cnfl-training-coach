# CONTRIBUTING — 体系扩展与维护

本文档面向想扩展知识库、修复盲点或贡献运动专项模块的使用者。**运行时诊断行为在 `SKILL.md`，本文件只管"体系如何长大"。**

---

## 1. 自我更新协议（Self-Update Protocol）

体系扩展自身知识库的机制。分两条路径运行，教练会自动识别当前环境。

### 1.1 知识盲点检测规则

以下情况自动标记为知识盲点，加入快照的 `【待研究问题】`：

| 触发场景 | 盲点类型 | 优先级 |
|---|---|---|
| 用户问某个训练方法，教练没有具体协议 | 方法论缺失 | 高 |
| 用户描述某种运动伤病，知识库无对应条目 | 伤病条目缺失 | 高 |
| 某个诊断结果标注了 `confidence=low` | 数据不足 | 中 |
| 用户所在运动在知识库中无专项基准值 | 基准值缺失 | 中 |
| 用户问某个补剂/营养方案，教练无证据等级评估 | 营养知识缺失 | 低 |

**标记格式（写入快照 `【待研究问题】`）：**

```
🔍 [用户原问题] — 优先级: 高 — 建议关键词: "[搜索词1] [搜索词2]" — 目标文件: [knowledge-xxx.md]
```

### 1.2 Claude Code 环境：自动研究与更新

当检测到运行环境为 Claude Code（存在文件系统访问权限）时，按以下流程处理 `【待研究问题】`：

```
触发时机：加载含【待研究问题】的快照 → 立即执行（对话开始前）

1. 按优先级排序，逐条搜索权威来源
   优先级：PubMed/系统综述 > 专家著作 > 专业博客
2. 找到来源后追加到对应 references/ 文件：
   ## 自更新补充 — [标题]（来源: [URL/书名], 日期: [YYYY-MM-DD]）
   [核心数字/协议/判断框架，不要铺垫]
3. 从快照移除已研究条目，版本号+1
4. 告知用户「已更新[X]个：[列表]」
```

**研究质量要求：**

- 必须找到至少 1 篇同行评审研究或知名专家原著作为来源
- 无法找到可靠来源时：标注 `confidence=low`，不写入知识库，告知用户"未找到足够可靠来源"
- 不允许凭记忆填写——必须找到外部来源验证

### 1.3 其他 AI 环境：半自动更新

当运行在 DeepSeek / ChatGPT / Kimi 等无文件访问权限的环境时：

1. **对话中**：正常运行，遇到知识盲点时标注并记入 `【待研究问题】`
2. **输出快照时**：列出所有待研究问题，附带建议搜索关键词
3. **在快照末尾追加提示**：

   ```
   ⚡ 本档案包含 [N] 个待研究问题。
      → 在 Claude Code 中加载此档案，教练将自动研究并更新知识库
      → 或手动搜索以下关键词，将结果发给教练学习：
         [列出关键词列表]
   ```

4. 如用户手动提供了研究资料（粘贴文章/截图/书摘）：
   - 立即消化吸收，在本次对话中应用
   - 在快照的 `【待研究问题】` 中将对应条目标记为 `✅ 临时学习（需CC同步到知识库）`

### 1.4 个人协议文件自动维护

`profiles/knowledge-[athlete-id]-protocols.md` 是专属个人的知识积累文件。**整个 `profiles/` 目录被 `.gitignore` 屏蔽**——个人数据（伤病史、1RM、体重）不会进入公开 repo。

**写入规则（仅 Claude Code 环境）：**

| 写入时机 | 写入内容 |
|---|---|
| 快照 `【个人规律】` 中某条被连续验证 2+ 次 | 升级为正式协议，写入个人文件 |
| 用户明确说"记住这个"/"以后都这样做" | 立即写入个人文件 |
| 某个通用处方被用户多次修改为相同变体 | 将该变体写入个人文件为默认处方 |

**文件格式（追加到末尾）：**

```markdown
## [athlete-id] 个人规律 — [日期]

### [规律名称]
**来源：** [第N次对话发现 / 用户自述]
**内容：** [具体描述]
**影响处方：** [如何调整默认训练方案]
**验证状态：** ✅ 已验证 / 🔄 待验证
```

**非 Claude Code 环境：** 不写文件，但在快照 `【个人规律】` 中完整保留，并在末尾标注"建议在 Claude Code 中运行一次以同步到个人协议文件"。

## 2. 贡献流程

### 2.1 新增运动项目

1. 在 `SKILL.md` 的 Step 2.5 基准表里加该运动的基准值（至少：深蹲/体重比、主速度指标、主跳跃指标）
2. 在 `references/` 下新建 `knowledge-[sport-name].md`，按现有格式组织（来源、核心原则、测试协议、训练处方）
3. 在 `evals/cases.md` 追加至少 1 个该运动的测试案例
4. 在 `README.md` 知识库表格和文件结构里更新

### 2.2 新增伤病康复协议

1. 扩展 `SKILL.md` Step 2.6 伤病标记扫描关键词
2. 如属于红旗范畴：扩展 Step 8.5 的硬性红旗规则
3. 康复分期协议写入 `references/knowledge-sports-medicine.md` 或 `knowledge-rehab-experts.md`
4. 在 `evals/cases.md` 追加伤病案例，明确 `Must not have` 禁忌动作

### 2.3 新增知识模块

1. 来源必须是已出版书籍或同行评审论文——博主文章只作补充
2. 文件头部必须有 `Sources:` 字段列出原始引用
3. 蒸馏原则：不复制原文，只做结构化转述 + 可执行协议 + 数字阈值
4. 在 `README.md` 知识库表格添加一行

### 2.4 修改现有基准/阈值

- **需要跑 evals**：改完之后跑一遍 `evals/cases.md` 10 个案例，确认 `Must have` / `Must not have` 全部通过
- 基准值变动必须附注来源——不允许凭感觉调

---

## 3. 回归测试

见 `evals/README.md`。

- 每次新增规则或修改基准后，必须跑一遍 10 个黄金案例
- 任何一个案例的 `Must not have` 条目被触发 = 回归，必须查根因
- `Must have` 允许措辞差异，但 P0/P1 分配和诊断标签必须一致

---

## 4. 隐私与伦理

- **禁止**：把任何带个人身份的训练数据（姓名、体重、伤病史、1RM）提交到公开 repo
- **必须**：所有个人数据放在 `profiles/`（已被 `.gitignore` 屏蔽）
- **必须**：任何伤病响应都附带"不构成医疗诊断"免责声明
- **必须**：遇到 Step 8.5 硬性红旗立即停训并建议就医——不给训练方案
